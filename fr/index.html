Labyrinth Path Solver

Labyrinth Path Solver est un projet conçu lors d'un cours d'algorithme et de structure de données. Ce projet permet de visualiser la génération et la résolution de labyrinthes, le tout à l'aide d'une interface graphique interactive.

Technologies utilisées : HTML, CSS, JavaScript (canvas, contexte 2D)

Projet de Bachelor

Le projet de bachelor, réalisé individuellement, consiste en deux parties distinctes :

  • Utilitaire permettant la génération de labyrinthes
  • Jeu en 3D permettant de parcourir des labyrinthes générés

L'utilitaire, inspiré du projet Labyrinth Path Solver décrit ci-dessus, permet de générer et de visualiser des labyrinthes composés de zones isolées appelées clusters (chaque cluster est représenté par une couleur distincte).

Les clusters sont ensuite reliés entre eux par des téléporteurs de manière à ce que chaque cluster soit accessible (chaque paire de téléporteurs est représentée par une lettre).

Un labyrinthe est composé de multiples salles, chaque salle ayant une difficulté qui y est associée (représenté par un nombre de 0 à 100), plus la salle est proche de l'arrivée, plus sa difficulté est élevée.

Les labyrinthes générés peuvent être enregistrés en tant que fichiers JSON. Un algorithme permet de déterminer le niveau de difficulté global d'un labyrinthe.

Le jeu en 3D permet de parcourir une succession de labyrinthes de difficulté croissante.

De multiples salles composées d'obstacles doivent être parcourues, les obstacles variant en fonction de la difficulté de la salle.

Le joueur peut en tout temps voir la carte du labyrinthe, permettant de connaitre sa localisation actuelle et l'emplacement des téléporteurs. Cependant, le joueur ne sait pas où mènent les téléporteurs. Il doit donc identifier la difficulté de la salle pour savoir s'il se rapproche ou s'éloigne de l'arrivée.

Le joueur dispose de trois points de vie. S'il tombe dans le vide, il perd une vie et réapparait à l'entrée de la salle actuelle. S'il perd tous ses points de vie, c'est game over ; le nombre de labyrinthes parcourus représente le score du joueur.

Technologies utilisées pour l'utilitaire : Python, Tkinter (interface graphique), JSON

Technologies utilisées pour le jeu en 3D : Godot (moteur de jeu), Blender (modélisation 3D), JSON

PixelARC

PixelARC est un langage de programmation conçu pour permettre la génération d'images dans le style "pixel art".

Le projet est composé de deux grandes parties :

  • Le compilateur (convertit le code PixelARC en bytecode)
  • L'interpréteur (interprète le bytecode et génère l'image finale)

Le langage implémente certains concepts tels que les variables, les conditions, les boucles, les fonctions, etc.
Des fonctions "natives" sont aussi implémentées, telles que des fonctions trigonométriques et des fonctions permettant de dessiner sur l'image finale.

Technologies utilisées : Python, Numpy, Pillow, Ply, PyDot

Arquizz

Arquizz est un site web inspiré de Kahoot! permettant de créer et de jouer à des quiz en ligne en affrontant d'autres utilisateurs en temps réel. Le projet implémente les fonctionnalités suivantes :

  • Gestion d'utilisateurs (inscription, connexion, profil)
  • Gestion des quiz (création, modification, suppression)
  • Gestion des questions et des propositions
  • Système de salons (partie publique pouvant être rejointe par n'importe qui via un code)
  • Système de points et de classement pour une partie
  • Système de temporisation pour une partie (limite de temps par question)
  • etc.

Technologies utilisées : HTML, CSS, JavaScript, PHP, Laravel, MySQL

Grafile

Grafile est un logiciel conçu pour automatiser certaines actions sur des fichiers, telles que les renommer, les déplacer, les supprimer, etc. L'interface graphique est sous la forme d'un système de nœuds pouvant être reliés entre eux.

Technologies utilisées : C++, Qt
